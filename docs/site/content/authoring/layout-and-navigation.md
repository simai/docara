# Макеты и навигация

Portable Docara использует два preset:

| Preset | Назначение |
| --- | --- |
| `docs` | Брендовая шапка, липкое дерево слева и область статьи |
| `landing` | Брендовая шапка и сфокусированная страница без дерева документации |

`layout.max_width` принимает `compact`, `normal`, `wide` или `full`.
`settings.theme` принимает `system`, `light` или `dark`.

Для меню доступны два параметра:

- `navigation.hidden` исключает страницу из глобального меню;
- `navigation.order` задаёт неотрицательный порядок; при равенстве используется стабильный путь источника;
- страница без `navigation.order` располагается после страниц с явно заданным порядком.

## Как строится дерево

Иерархия берётся из каталогов и Markdown-файлов, а не из `slug`. Заголовок
пункта берётся из page sidecar, первого Markdown H1 или `_section.json` для
виртуального каталога.

```text
content/
  guides.md                         # уровень 1 и ссылка на обзор
  guides/
    install.md                      # уровень 2
    configuration.md                # уровень 2 и ссылка на обзор
    configuration/
      theme.md                      # уровень 3
      advanced/
        overrides.md                # уровень 4
```

Страница `guides.md` и одноимённый каталог `guides/` становятся одним пунктом:
текст открывает обзор, отдельная кнопка раскрывает потомков. Вместо
`guides.md` можно использовать `guides/index.md`. Каталог без страницы
остаётся понятной группой без ложной ссылки.

Внутренняя модель не ограничивает глубину. Интерфейс и Simai Framework
проверены на четырёх видимых уровнях; более глубокие элементы сохраняют
семантику и используют визуальное оформление четвёртого уровня. Посмотрите
[живую страницу четвёртого уровня](/authoring/layout-and-navigation/hierarchy/four-level/).

## Активное состояние и мобильный экран

Текущая ссылка получает `aria-current="page"`, а все её предки раскрываются.
На широком экране дерево закреплено слева и прокручивается независимо. До
800 px оно скрывается и доступно через компактный пункт «Разделы» в шапке.
Escape закрывает мобильную область и возвращает фокус к её заголовку.

Ссылки остаются нативными и работают с Enter. Кнопка раскрытия управляет только
веткой, поэтому раскрытие не мешает переходу на страницу-раздел.
