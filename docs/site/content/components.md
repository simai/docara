# Компоненты

Docara выбирает возможности из трёх исполняемых семейств:

1. native Markdown для текста, ссылок, списков, кода и таблиц;
2. typed-компоненты Docara для проверяемой семантической композиции;
3. Smart-компоненты Simai Framework, допущенные точным Framework lock.

## Где смотреть точные сведения

- [Живой каталог компонентов](/components/catalog/) — доступность, пример и
  ограничения конкретной сборки.
- [Синтаксис и выбор семейства](/components/syntax/) — общие правила вызова.

Generated catalog является единственным детальным справочником. Он строится из
владеющих JSON-записей, manifests и исполняемых example fixtures. Эта страница
не повторяет параметры, состояния и ограничения отдельных компонентов.

Сборка также записывает machine-readable проекцию:

```text
_docara/component-catalog.json
```

Публичный URL учитывает `base_url`.

## Lifecycle

- `supported` означает, что для записи есть renderer, tests, docs и
  исполняемый example; Smart-возможность также допущена точным lock.
- `admission_pending`, `framework_gap` и `deferred` описывают недоступную
  потребность.

Недоступная запись объясняет owner, причину, fallback и условие допуска, но не
создаёт callable syntax и отдельную detail-page.

## Как выбирать

1. Используйте native Markdown, если он выражает задачу.
2. Выберите typed-компонент для семантического блока Docara.
3. Выберите Smart-компонент, если нужен его подтверждённый UI contract.
4. Если нужной возможности нет в generated catalog, используйте указанный
   fallback или оформите requirement; не вставляйте произвольный HTML.

Каталог не является вторым реестром Simai Framework и не может сам разрешить
компонент. Фактический допуск всегда выводится из immutable inputs.
