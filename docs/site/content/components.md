# Компоненты

Docara собирает один эффективный каталог из трёх исполняемых семейств:

- **Native Markdown** — заголовки, текст, ссылки, изображения, списки, цитаты,
  код и таблицы из ограниченного переносимого профиля;
- **typed-компоненты Docara** — безопасные семантические блоки `card`, `steps`,
  `cta` и `features` с собственными проверяемыми renderer;
- **Smart-компоненты Simai Framework** — только компоненты, допущенные точным
  `simai-framework.lock.json`, с manifest, проверкой props и зафиксированными
  ассетами.

Сборка сохраняет эту производную проекцию в
`_docara/component-catalog.json` (публичный URL учитывает `base_url`) как
`docara.effective_component_catalog.v1`. Это не второй реестр Simai Framework:
каталог не может сам разрешить Smart-компонент и не заявляет готовность всех
компонентов или production readiness.

У каждой записи есть lifecycle: `supported`, `admission_pending`,
`framework_gap` или `deferred`. Неисполняемые записи потребностей объясняют,
чего не хватает, кто владеет изменением, какой есть fallback и что должно
произойти для допуска. Наличие такой записи не делает синтаксис доступным.

## Справочник

- [Живой каталог компонентов](/components/catalog/)
- [Синтаксис и границы](/components/syntax/)

Живой каталог генерируется из эффективного каталога конкретной сборки. Он
показывает точный авторский синтаксис, отрисованный пример, параметры,
состояния, ограничения и источник каждой поддерживаемой возможности. Для
недоступных потребностей индекс объясняет gap, fallback и условие будущего
допуска.

## Адаптивные колонки

Компонент `docara.columns` размещает от двух до четырёх областей Markdown в
адаптивной сетке Simai Framework. На широком экране области становятся
колонками, на узком — сохраняют исходный порядок и складываются вертикально.

```markdown
:::columns
### Первая область

Содержание первой области.

---

### Вторая область

Содержание второй области.
:::
```

Области разделяются отдельной строкой `---` с пустыми строками вокруг.
Вложенные typed-компоненты Docara и Smart-компоненты внутри колонок пока не
поддерживаются. Актуальный живой пример и точные ограничения находятся на
странице [`docara.columns`](/components/catalog/docara.columns/).

Машинным источником фактического статуса остаётся каталог конкретной сборки.
Не используйте произвольный HTML вместо отсутствующего компонента: raw HTML в
переносимом Markdown удаляется.
