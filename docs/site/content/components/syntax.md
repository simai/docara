# Синтаксис и границы компонентов

Smart-компонент вызывается fenced-директивой с одним JSON-объектом props:

```markdown
:::ui.alert
{"type":"info","variant":"default","title":"Готово","supporting-text":"Страница собрана.","closable":false,"aria-label":"Страница готова"}
:::
```

Вызов проходит только если идентификатор есть в versioned contract, manifest
совпадает с Framework lock, все props имеют допустимый тип и соблюдены
ограничения компонента. Неизвестный компонент или prop останавливает сборку.

`:::card`, `:::steps`, `:::cta` и `:::features` не являются
Smart-компонентами: это безопасные семантические Markdown-блоки, оформленные
утилитами Simai Framework.

Не вкладывайте `card`, `steps`, `cta`, `features` и `ui.*` друг в друга: любая
такая комбинация останавливает сборку с ошибкой nesting. Один блок `steps`
должен содержать ровно один нумерованный Markdown-список. `cta` принимает одну
безопасную Markdown-ссылку, а `features` — один плоский маркированный список
из двух–шести пунктов, по одному обычному Markdown-абзацу в каждом. Все
директивы начинаются без отступа на верхнем уровне документа; директива внутри
списка отклоняется, чтобы содержимое не переместилось в другой
CommonMark-контейнер.
Если внутри блока нужно показать строку, похожую на открытие другой директивы,
поместите пример в обычный fenced code block.

Пустая строка перед закрывающим `:::` не обязательна. Для совместимости
закрывающий fence может иметь до трёх начальных пробелов, но открывающий fence
всегда начинается с первой позиции строки и не прерывает обычный абзац.

Если содержимое само должно включать отдельную строку `:::`, используйте более
длинный внешний fence. Закрывающий fence должен быть не короче открывающего:

```markdown
::::card
Эта строка остаётся содержимым:
:::
И эта тоже.
::::
```

Такой же приём применяйте для standalone-разделителя внутри raw HTML,
многострочного inline-фрагмента или reference title. Обычные fenced code blocks
распознаются CommonMark и не закрывают внешнюю директиву преждевременно.

Одна страница может содержать не больше 64 строк, похожих на открытие
директивы. Ограничение проверяется до извлечения блоков и защищает сборку от
неограниченного повторного разбора сгенерированного Markdown.
