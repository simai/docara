# Контекст чтения

Контекст чтения помогает понять три вещи: где находится страница, что есть
внутри неё и куда перейти дальше. Docara строит его автоматически из структуры
Markdown-проекта и заголовков статьи.

## Что появляется на странице

Для preset `docs` доступны четыре настройки:

| Поле | Значение по умолчанию | Что делает |
| --- | --- | --- |
| `reading.breadcrumbs` | `true` | Показывает путь от главной до текущей страницы |
| `reading.toc` | `true` | Показывает оглавление по заголовкам статьи |
| `reading.toc_depth` | `3` | Включает заголовки от H2 до указанного уровня, допустимо 2–6 |
| `reading.previous_next` | `true` | Показывает соседние страницы в порядке дерева |

На широком экране оглавление закреплено справа. Ниже 1152 px оно становится
раскрываемым блоком «На этой странице» над статьёй. На узком экране переходы
«Предыдущая» и «Следующая» складываются друг под другом.

Preset `landing` не выводит хлебные крошки, оглавление и переходы: эти элементы
относятся к чтению документации, а не к устройству лендинга.

## Настройка сайта и раздела

Обычно достаточно включить контекст один раз в `docara.json`:

```json
{
  "schema": "docara.site.v1",
  "reading": {
    "breadcrumbs": true,
    "toc": true,
    "toc_depth": 3,
    "previous_next": true
  }
}
```

Раздел может изменить только нужное поле и сохранить остальные значения:

```json
{
  "schema": "docara.section.v1",
  "reading": { "toc_depth": 4 }
}
```

## Настройка одной страницы

Чтобы оставить стабильные ссылки на заголовки, но убрать все вспомогательные
поверхности, добавьте в `<page>.page.json`:

```json
{
  "schema": "docara.page.v1",
  "reading": {
    "breadcrumbs": false,
    "toc": false,
    "previous_next": false
  }
}
```

Docara всё равно назначит заголовкам детерминированные `id`. При неизменном
тексте и порядке заголовков ссылка на раздел статьи продолжит работать, даже
если оглавление скрыто. Переименование заголовка меняет автоматически
сформированный якорь и требует проверки входящих deep links.

## Как формируются якоря

Все заголовки H1–H6 получают детерминированный Unicode-якорь. Регистр
приводится к нижнему, пробелы и знаки пунктуации превращаются в дефис:

```text
## Установка и настройка
→ #установка-и-настройка
```

Если одинаковый заголовок повторяется, Docara добавляет суффикс `-1`, `-2` и
далее. Заголовок без букв и цифр получает основу `section`. В оглавление входят
только H2–H6 в пределах `toc_depth`; H1 остаётся заголовком страницы.

В Markdown можно ссылаться на получившийся раздел обычной локальной ссылкой:

```markdown
[Перейти к настройке](#установка-и-настройка)
```

Статическая проверка сборки отклонит ссылку на отсутствующий фрагмент и страницу
с повторяющимся HTML `id`.

## Порядок страниц

Хлебные крошки и переходы используют единую каноническую топологию:

1. структура каталогов задаёт вложенность;
2. `navigation.order` задаёт порядок соседей;
3. при равенстве применяется стабильный путь источника;
4. страницы другого языка и preset `landing` не становятся соседями документа;
5. скрытая страница остаётся частью пути, но не становится целью перехода.

`navigation.hidden: true` скрывает страницу только из глобального меню. Если у
скрытой страницы есть видимые потомки, ветка остаётся в меню как группа без
ложной ссылки. Благодаря общей топологии хлебные крошки скрытой страницы не
теряют её реальный путь, а переходы находят ближайших видимых соседей.

Главная страница не показывает хлебные крошки из единственного пункта. Страница
только с H1 не показывает пустое оглавление, но сам H1 всё равно получает
якорь.

## Когда отключать элементы

- Отключите `toc`, если страница короткая и состоит только из одного заголовка.
- Уменьшите `toc_depth`, если глубокие технические подпункты перегружают правую
  область.
- Отключите `previous_next` для автономной справочной страницы, где порядок
  чтения не важен.
- Отключите `breadcrumbs` только если путь уже однозначно объяснён самой
  страницей; левое дерево при этом остаётся доступным.

Фактические итоговые значения и их источники можно проверить в
[resolved page plan](/reference/resolved-plan/).
