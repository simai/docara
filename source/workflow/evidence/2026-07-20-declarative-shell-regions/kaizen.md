# Kaizen

1. Product composites must use the shared Larena manifest vocabulary without
   pretending to be native Simai Framework web components.
2. Shell compilation must happen after canonical navigation activation and
   branding resolution; compiling during page discovery loses real state.
3. Content semantic parity and shell structural parity are different checks.
   Keeping them separate makes failures local and prevents shell links from
   contaminating article semantics.
4. A bounded flattened navigation view model keeps templates
   presentation-only while preserving explicit depth and active-path data.
5. The current broad Pint command sees the accepted `ui.alert` template's
   alternative PHP syntax. Changed PHP implementation and tests were checked
   with a scoped Pint gate; no unrelated template rewrite was introduced.
