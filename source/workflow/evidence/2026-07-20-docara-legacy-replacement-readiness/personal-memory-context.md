# Personal memory context

Decision: `inject`

## Reused orientation

- Docara source repositories and local ServBay sites have separate
  source/runtime boundaries.
- `docara.test` changes require a fresh backup, staging build, rollback path
  and served HTTP/browser proof.
- The retained `docara-legacy.test` layout was rebuilt from legacy main source
  and is a behavioral/visual reference, not the implementation target.
- ServBay PHP should be addressed explicitly when the shell PHP toolchain is
  affected by Homebrew ICU drift.

## Verification policy

Memory-derived context is orientation only. Before every implementation or
acceptance claim, verify current:

- current Git branch, HEAD and worktree status;
- current portable source/tests/schemas and exact Framework lock;
- current legacy route/source inventory;
- current `docara.test` and `docara-legacy.test` responses;
- current ServBay/PHP/runtime state before local publication.

Do not reuse old build counts, hashes, screenshots, backup paths or runtime
status as current evidence.
